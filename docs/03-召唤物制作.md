## 基础概念

### 召唤物与敌人的关系
- 召唤物和敌人都继承自 `MonsterModel`
- 召唤物通过 `PetOwner` 属性关联到玩家
- 召唤物属于玩家阵营（`Side = Player`）

### 核心类
- `MonsterModel` - 怪物/召唤物基类
- `Creature` - 生物实体类
- `PlayerCmd.AddPet<T>()` - 召唤命令

```csharp
        return NodeFactory<NCreatureVisuals>.CreateFromResource(texture);
    }
}
```

## 召唤机制

### 召唤流程
1. 调用 `PlayerCmd.AddPet<T>()`
2. 创建 Creature 实体
3. 设置 `PetOwner` 属性
4. 添加到玩家的宠物列表

### 召唤代码示例
```csharp
// 在能力中召唤
public class BirthPower : JoiPower, IOnPowerRemoved
{
    public async Task OnPowerRemoved(PowerChangeContext context)
    {
        // 召唤周心
        var creature = await SummonActions.SummonAlly(
            SummonDefinition.For<ZhouXin>(uniqueKey: "zhou-xin-core"),
            Owner);
    }
}
```

### 在卡牌中召唤
```csharp
protected override async Task OnPlay(PlayerChoiceContext choiceContext, CardPlay cardPlay)
{
    // 召唤召唤物
    await PlayerCmd.AddPet<ZhouXin>(Owner);
}
```

### 原版召唤命令
```csharp
// 原版游戏的 Osty 召唤
await OstyCmd.Summon(choiceContext, Owner);

// 通用宠物召唤
await PlayerCmd.AddPet<T>(player);
```

### 创建自定义视觉

```csharp
public override NCreatureVisuals CreateCustomVisuals()
```

### 图片资源要求
- **尺寸**: 200 × 200 px（推荐）
- **格式**: PNG 8-bit/color RGBA
- **路径**: `Joi/images/creatures/{name}.png`

### 动画状态
```csharp
// 默认状态
public override string? DefaultMoveState => "idle";

// 可用状态：
// - "idle" - 待机
// - "attack" - 攻击
// - "hurt" - 受伤
// - "death" - 死亡
```

## 行动系统

### 无行动召唤物
```csharp
// 类似 Osty，不执行任何行动
public override MonsterMoveStateMachine CreateMoveStateMachine()
{
    return new MonsterMoveStateMachine(
        new MonsterMove("NOTHING", [])
    );
}
```

### 有行动召唤物
```csharp
public override MonsterMoveStateMachine CreateMoveStateMachine()
{
    return new MonsterMoveStateMachine(
        // 攻击行动
        new MonsterMove("ATTACK", [
            new AttackIntent(8)
        ]),
        // 防御行动
        new MonsterMove("DEFEND", [
            new BlockIntent(5)
        ])
    );
}
```

## 特殊机制

### 1. 伤害转向
```csharp
// 将对玩家的伤害转向到召唤物
public override Creature ModifyUnblockedDamageTarget(
    Creature target,
    decimal amount,
    ValueProp props,
    Creature? dealer)
{
    // 如果目标是玩家，且伤害来自敌方
    if (target.Side == Creature.Side &&
        !Creature.IsDead &&
        dealer?.Side != Creature.Side)
    {
        return Creature;  // 转向到召唤物
    }
    return target;
}
```

### 2. 时间限制
```csharp
// 在召唤时应用时间限制能力
public async Task OnSummon(PlayerChoiceContext choiceContext)
{
    // 3 回合后自动消失
    await PowerCmd.Apply<TimeLimitPower>(Creature, 3);
}
```

### 3. 死亡时触发
```csharp
public override async Task OnDeath(PlayerChoiceContext choiceContext)
{
    // 召唤物死亡时的效果
    await CreatureCmd.Damage(choiceContext,
        CombatState?.Enemies.ToList() ?? [],
        10, ValueProp.Unpowered, Creature, null);
}
```

### 4. 回合开始/结束
```csharp
public override async Task OnTurnStart(PlayerChoiceContext choiceContext)
{
    // 回合开始时的效果
    await CreatureCmd.Heal(choiceContext, Creature, 1);
}

public override async Task OnTurnEnd(PlayerChoiceContext choiceContext)
{
    // 回合结束时的效果
}
```

## 完整示例

### 周心（ZhouXin）召唤物
```csharp
// 路径：JoiCode/Minions/ZhouXin.cs
using BaseLib.Abstracts;
using MegaCrit.Sts2.Core.Models;
using MegaCrit.Sts2.Core.Entities.Creatures;
using MegaCrit.Sts2.Core.Entities;
using Godot;

namespace Joi.Minions;

public class ZhouXin : CustomMonsterModel
{
    public override string? DefaultMoveState => "idle";
    public override int MinInitialHp => 1;
    public override int MaxInitialHp => 1;
    public override bool CanBeTargeted => true;

    public override NCreatureVisuals CreateCustomVisuals()
    {
        Texture2D texture = GD.Load<Texture2D>(
            "res://Joi/images/creatures/zhou_xin.png");
        return NodeFactory<NCreatureVisuals>.CreateFromResource(texture);
    }

    public override MonsterMoveStateMachine CreateMoveStateMachine()
    {
        // 不执行任何行动
        return new MonsterMoveStateMachine(
            new MonsterMove("NOTHING", [])
        );
    }

    // 伤害转向：将对玩家的伤害转向到周心
    public override Creature ModifyUnblockedDamageTarget(
        Creature target,
        decimal amount,
        ValueProp props,
        Creature? dealer)
    {
        if (target.Side == Creature.Side &&
            !Creature.IsDead &&
            dealer?.Side != Creature.Side)
        {
            return Creature;
        }
        return target;
    }

    // 周心死亡时的效果
    public override async Task OnDeath(PlayerChoiceContext choiceContext)
    {
        // 对所有敌人造成伤害
        await CreatureCmd.Damage(choiceContext,
            Creature.CombatState?.Enemies.ToList() ?? [],
            5, ValueProp.Unpowered, Creature, null);
    }
}
```

## 原版召唤物分析

### Osty
- **HP**: 1（固定）
- **行动**: 无（NOTHING_MOVE）
- **特点**:
  - 专属 `TargetType.Osty`
  - 特殊的动画和视觉效果
  - 通过 `OstyCmd.Summon()` 创建

### BattleFriend V1/V2/V3
- **HP**: 75 / 150 / 300
- **行动**: 无（NOTHING_MOVE）
- **特点**:
  - 应用 `BattlewornDummyTimeLimitPower`（3 回合）
  - 作为肉盾使用

## 最佳实践

1. **生命值设计**
   - 肉盾型：高生命值（50+）
   - 功能型：低生命值（1-10）
   - 平衡型：中等生命值（20-40）

2. **行动设计**
   - 功能型召唤物：无行动（NOTHING_MOVE）
   - 战斗型召唤物：有攻击/防御行动
   - 考虑回合限制（TimeLimitPower）

3. **视觉效果**
   - 使用清晰的图标
   - 尺寸：200 × 200 px
   - 与人物主题协调

4. **特殊机制**
   - 伤害转向：保护玩家
   - 死亡触发：额外效果
   - 回合触发：持续效果

5. **平衡性**
   - 召唤成本要合理
   - 召唤物强度要适中
   - 考虑与其他卡牌的配合
