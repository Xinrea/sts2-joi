## Atlas 系统

### 什么是 Atlas
Atlas（图集）是将多个小图像打包成一个大图像的技术，用于优化性能。

### Atlas 路径格式
```
res://images/atlases/{atlas_name}.sprites/{sprite_name}.tres
```

### 常用 Atlas
| Atlas 名称 | 用途 | 路径模式 |
|-----------|------|---------|
| `relic_atlas` | 遗物图标 | `atlases/relic_atlas.sprites/{id}.tres` |
| `relic_outline_atlas` | 遗物轮廓 | `atlases/relic_outline_atlas.sprites/{id}.tres` |
| `potion_atlas` | 药水图标 | `atlases/potion_atlas.sprites/{id}.tres` |
| `potion_outline_atlas` | 药水轮廓 | `atlases/potion_outline_atlas.sprites/{id}.tres` |

## 资源加载机制

### 资源加载方式

#### 1. 直接加载
```csharp
Texture2D texture = GD.Load<Texture2D>("res://Joi/images/card_portraits/strike.png");
```

#### 2. 通过 ResourceLoader
```csharp
Texture2D texture = ResourceLoader.Load<Texture2D>(
    path,
    null,
    ResourceLoader.CacheMode.Reuse);
```

#### 3. 通过 PreloadManager
```csharp
Texture2D texture = PreloadManager.Cache.GetTexture2D(path);
```

## 格式要求

### 图片格式
- **格式**: PNG
- **色彩**: 8-bit/color RGBA
- **压缩**: non-interlaced
- **透明度**: 支持 Alpha 通道

### 文件大小
- **卡牌图像**: < 500 KB
- **人物图像**: < 1 MB
- **遗物/药水图标**: < 100 KB
- **怪物图像**: < 200 KB

### 优化建议
1. **使用 PNG 优化工具**
   - OptiPNG
   - PNGCrush
   - TinyPNG

2. **调整图像尺寸**
   ```bash
   # 使用 ImageMagick 调整尺寸
   convert input.png -resize 256x256 output.png
   ```

3. **检查文件大小**
   - 如果文件过大（> 1MB），考虑降低分辨率
   - 使用 `resize_image.ps1` 脚本调整大小

## 常见问题

### 1. 图像不显示
**原因**:
- 文件名不匹配（大小写、下划线）
- 路径错误
- 文件格式不正确

**解决方案**:
```csharp
// 检查文件是否存在
if (ResourceLoader.Exists(path))
{
    // 加载图像
}
else
{
    // 使用默认图像
}
```

### 2. Godot 显示 "Not a PNG file"
**原因**: PNG 文件损坏或格式不正确

**解决方案**:
1. 删除 `.import` 文件
2. 重新编译项目
3. 使用 PNG 优化工具修复文件

### 3. 图像过大导致性能问题
**原因**: 图像尺寸或文件大小过大

**解决方案**:
1. 调整图像尺寸到推荐大小
2. 使用 PNG 优化工具压缩
3. 考虑使用 Atlas 系统

## 最佳实践

1. **尺寸标准化**
   - 严格遵守推荐尺寸
   - 使用 2 的幂次方尺寸（256, 512, 1024）
   - 保持宽高比

2. **文件组织**
   - 按类型分类存放
   - 使用一致的命名规范
   - 保持目录结构清晰

3. **性能优化**
   - 压缩 PNG 文件
   - 使用 Atlas 系统
   - 预加载常用资源

4. **版本控制**
   - 不要提交过大的图像文件
   - 使用 Git LFS 管理大文件
   - 定期清理未使用的资源

5. **测试验证**
   - 在游戏中测试图像显示
   - 检查不同分辨率下的效果
   - 验证透明度和边缘
