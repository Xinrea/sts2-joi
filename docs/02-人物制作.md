## 基础概念

### 人物系统架构
- `PlaceholderCharacterModel` - 占位符人物基类（用于模组）
- `CharacterModel` - 原版人物基类
- 每个人物有独立的卡牌池、遗物池、药水池

### 人物 ID
人物 ID 是唯一标识符，用于：
- 本地化文件路径
- 资源文件路径
- 存档数据

### 基本结构
```csharp

public class Joi : PlaceholderCharacterModel

        ModelDb.Card<Strike>(),
        ModelDb.Card<Strike>(),
        ModelDb.Card<Strike>(),
        ModelDb.Card<Defend>(),
        ModelDb.Card<Defend>(),
        ModelDb.Card<Defend>(),
        ModelDb.Card<Defend>(),
        ModelDb.Card<Teleport>(),
        ModelDb.Card<BlackHoleStrike>()
    ];

    public override PotionPoolModel PotionPool => ModelDb.PotionPool<JoiPotionPool>();
}
```

## 卡牌池配置

### 卡牌池类
```csharp
// 路径：JoiCode/Character/JoiCardPool.cs
public class JoiCardPool : CustomCardPoolModel
{
    public override string Title => Joi.CharacterId;

    // 能量图标（大）
    public override string BigEnergyIconPath => "charui/big_energy.png".ImagePath();

    // 能量图标（文本）
    public override string TextEnergyIconPath => "charui/text_energy.png".ImagePath();

    // HSV 颜色调整
    public override float H => 0.12f;  // 色调 (0-1)
    public override float S => 1f;     // 饱和度 (0-1)
    public override float V => 1f;     // 亮度 (0-1)

    // 卡组条目颜色
    public override Color DeckEntryCardColor => Joi.Color;

    // 是否为无色卡池
    public override bool IsColorless => false;
}
```

### HSV 颜色说明
- **H (Hue)**: 色调，0-1 对应 0°-360°
  - 0.0 = 红色
  - 0.12 = 橙黄色
  - 0.33 = 绿色
  - 0.66 = 蓝色
- **S (Saturation)**: 饱和度，0 = 灰色，1 = 纯色
- **V (Value)**: 亮度，0 = 黑色，1 = 最亮

## 最佳实践

1. **生命值平衡**
   - 考虑人物机制的风险性
   - 高风险机制 = 低生命值
   - 稳定机制 = 高生命值

2. **初始卡组设计**
   - 4-5 张攻击 + 4-5 张防御
   - 1-2 张特色卡牌展示机制
   - 总计 10 张卡牌

3. **颜色主题**
   - 选择独特的主题色
   - HSV 调整要与主题色协调
   - 能量图标颜色要匹配

4. **资源组织**
   - 所有 UI 资源放在 `charui/` 目录
   - 使用扩展方法获取路径
   - 保持命名一致性
